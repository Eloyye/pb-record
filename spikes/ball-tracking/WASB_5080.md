# Running the real ball tracker on the 5080 box

`spike.py` is tracker-agnostic — it just needs a `frame,x,y,visible` CSV. This runbook
produces that CSV from a real tracker on your Linux RTX 5080 machine; you can then run
`spike.py --predictions …` here or there.

## Why this exact combination

- **WASB** (nttcom/WASB-SBDT) has the best weights for us — a *tennis* model
  (`wasb_tennis_best.pth.tar`), the closest public match to a pickleball ball — but it is a
  benchmark repo with **no custom-video inference path**.
- **BlurBall** (cogsys-tuebingen/blurball) is a WASB-derived repo that *does* take a raw video:
  `main.py --config-name=inference_wasb … +input_vid=clip.mp4`. Its own weights are table-tennis
  (poor transfer), but it runs the same WASB architecture.
- So: **BlurBall's harness + WASB's tennis checkpoint.** Same architecture → the checkpoint
  should load.

## Two things that will likely need attention on first run

1. **Torch vs Blackwell.** The RTX 5080 is Blackwell (sm_120). BlurBall pins `torch==2.2.2`, which
   predates Blackwell and lacks the CUDA kernels for it. `run_wasb_5080.sh` installs BlurBall's deps
   *minus* torch, then a CUDA 12.8 torch build. If `timm`/`lightning` complain about the newer torch,
   that's the knob to fiddle.
2. **Output → CSV mapping.** BlurBall's per-frame output columns aren't documented. `wasb_to_csv.py`
   assumes a TrackNet-style `Frame,Visibility,X,Y`; if BlurBall names them differently, pass
   `--frame-col/--x-col/--y-col/--vis-col`, or eyeball one row and adjust.

> Authored on a Mac — I could **not** execute this. Treat the two items above as the first-run
> friction; everything else is grounded in the repos' actual docs and pinned deps.

## Steps

```bash
# on the 5080 (Linux) box:
bash run_wasb_5080.sh /path/to/live_play_clip.mp4
# → prints where BlurBall wrote its per-frame output, then (wasb_to_csv.py is in this spike dir):
python wasb_to_csv.py <blurball_output.csv> --out predictions.csv
# → then, here or there:
uv run spike.py --video /path/to/live_play_clip.mp4 --predictions predictions.csv
```

Read the result against the rubric in `README.md`.

## Fallbacks

- If the WASB tennis checkpoint won't load into BlurBall's `inference_wasb` (arch mismatch), fall
  back to BlurBall's bundled **WASB table-tennis** weights just to get a run — and judge with the
  scale mismatch in mind.
- If BlurBall is fiddly, `qaz812345/TrackNetV3` ships a first-class `predict.py` (video → CSV) with
  badminton weights; swap it in — same CSV contract, then the same `wasb_to_csv.py` step.
